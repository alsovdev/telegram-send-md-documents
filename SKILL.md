# Send MD Documents to Telegram

## When to use

When the user asks to send, deliver, or present Markdown (.md) files to a Telegram chat. The built-in `send_message` tool with `MEDIA:` parameter does NOT send files as documents — it auto-detects them as photo/video/audio and renders them inline. This skill uses direct Telegram Bot API to send actual downloadable document files.

## Prerequisites

- Telegram Bot Token stored in `~/.hermes/.env` as `TELEGRAM_BOT_TOKEN`
- Target chat ID stored in `~/.hermes/.env` as `TELEGRAM_HOME_CHANNEL`
- Bot already connected via Hermes gateway

## Method: curl multipart POST to sendDocument

Use Python script to send file as document via Telegram Bot API:

```python
import urllib.request, json

# Read token
token = None
with open('/home/usr/.hermes/.env') as f:
    for line in f:
        if 'TELEGRAM_BOT_TOKEN' in line and not line.startswith('#'):
            token = line.strip().split('=', 1)[1].strip()
            break

boundary = '----WebKitFormBoundary7MA4YWxkTrZu0gW'
file_path = '/path/to/file.md'  # Replace with actual path
filename = 'document.md'          # Replace with desired filename (Russian OK)

# Read chat_id from .env (same as token)
import os
env_path = os.path.expanduser('~/.hermes/.env')
chat_id = None
with open(env_path) as f:
    for line in f:
        if 'TELEGRAM_HOME_CHANNEL' in line and not line.startswith('#'):
            chat_id = line.strip().split('=', 1)[1].strip()
            break
if not chat_id:
    chat_id = None

with open(file_path, 'rb') as f:
    file_data = f.read()

body = b''
body += ('--' + boundary + '\r\n').encode()
body += b'Content-Disposition: form-data; name="chat_id"\r\n\r\n'
body += (chat_id + '\r\n').encode()
body += ('--' + boundary + '\r\n').encode()
body += ('Content-Disposition: form-data; name="document"; filename="' + filename + '"\r\n').encode()
body += b'Content-Type: text/markdown\r\n\r\n'
body += file_data + b'\r\n'
body += ('--' + boundary + '--\r\n').encode()

url = f"https://api.telegram.org/bot{token}/sendDocument"
req = urllib.request.Request(url, data=body, method='POST')
req.add_header('Content-Type', f'multipart/form-data; boundary={boundary}')

with urllib.request.urlopen(req) as resp:
    data = json.loads(resp.read())
    if data.get('ok'):
        print(f"OK: {filename} sent as document")
    else:
        print(f"FAIL: {data}")
```

## Key details

- **Content-Type**: `text/markdown` (not `text/plain`)
- **Filename**: Can be in Cyrillic, with spaces, must end in `.md`
- **File size limit**: Telegram Bot API limit is 2 GB for documents
- **Token location**: `~/.hermes/.env` → `TELEGRAM_BOT_TOKEN=...`
- **chat_id**: `~/.hermes/.env` → `TELEGRAM_HOME_CHANNEL=...`

## Why not `send_message` with `MEDIA:`?

`send_message` with `MEDIA:/path` auto-detects file type:
- Images → renders as photo
- Videos → renders as video
- Audio → renders as voice message
- Everything else → may render as text or fail

It does NOT send as a downloadable document with `sendDocument` endpoint. The curl multipart method is required for proper document delivery.

## Token handling — CRITICAL

- **Never paste secrets inline in agent code** — terminal/shell heredoc rendering truncates or masks them (>20 chars often becomes `***` or loses middle characters)
- **Write token to file first**: have user send token as a FILE (not paste in chat) or write via `echo "TOKEN" > /tmp/token.txt` then read in Python
- **Tokens in Telegram messages get truncated**: when users paste long tokens/URLs in Telegram chat, the message may be visually truncated by Telegram client even though the full text is present on server side. If a token/auth string looks too short, ask user to send it as a file
- **Verify length**: always `print(f"Token length: {len(token)}")` before use — GitHub PAT should be 40 chars, Telegram bot token ~46 chars

## Pitfalls

1. **Long strings truncated in Telegram messages**: not just URLs — tokens, base64, hashes, and any string >~35 chars may appear cut off in the Telegram client. Always verify by asking user to confirm or send as file.
2. **Token masked in logs**: The token may appear as `***` in terminal output. Use Python to read it programmatically.
3. **Token truncated in heredoc/echo**: When passing tokens via bash heredoc or echo, special characters or length may cause truncation. Always write tokens to a file first then read in Python.
4. **Bytes + string mixing**: In the multipart body, use `.encode()` for string parts and `b''` for binary parts.
5. **Filename encoding**: Cyrillic filenames work fine in the `Content-Disposition` header.
6. **One request per file**: Don't batch multiple files in one multipart request — send each separately.
7. **`send_message` MEDIA: does NOT work for documents**: The built-in `send_message` tool auto-detects file types and renders as photo/video/audio. It cannot send arbitrary files as downloadable documents. Always use this skill's method for document delivery.
8. **Always verify token length**: `print(f"Token length: {len(token)}")` before use — GitHub PAT = 40 chars, Telegram bot token = ~46 chars. If much shorter, truncation happened.
9. **User preference: when user asks to "send md files" — use this curl/multipart method, NOT `send_message`**. The user explicitly requested this method after MEDIA: failed to deliver documents properly.
